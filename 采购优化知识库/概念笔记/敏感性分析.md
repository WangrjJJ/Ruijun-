# 敏感性分析

> [!abstract] 定义
> 在最优解确定后，通过==系统性地扰动关键参数==，观察目标函数和决策方案的变化幅度，评估方案的**鲁棒性**（Robustness）。对应[[Rayray课代表的数字化管理(8) - 预测那些事儿3 - 机器学习结合运筹优化|预测3]]中"预测质量对运筹优化的影响"的讨论。

## 为什么需要敏感性分析？

> [!question]- 最优解一定可靠吗？
> 不一定。最优解基于输入参数（价格、需求、MOQ等），而这些参数本身存在不确定性：
> - 供应商报价可能波动
> - 客户需求可能调整
> - MOQ 门槛可能谈判调整
>
> 如果参数微小变化就导致方案剧烈变化，说明方案==不够鲁棒==，需要谨慎决策。

## 三维敏感性设计

在[[钢材采购优化模型]]中，设计了 **3 个维度 × 19 个场景**：

| 维度 | 参数 | 扰动范围 | 业务含义 |
|------|------|----------|----------|
| 维度1 | 非标价格溢价 | 0% ~ +20% | 供应商谈判空间 |
| 维度2 | 非标 MOQ 门槛 | 4吨 ~ 12吨 | 起订量弹性 |
| 维度3 | 需求缩放因子 | 0.7x ~ 1.3x | 市场不确定性 |

### 19 个场景矩阵

```
基准场景（1个）：当前参数
溢价扰动（6个）：0%, 2%, 5%, 8%, 10%, 15%, 20%
MOQ扰动（6个）：4t, 5t, 6t, 8t, 10t, 12t
需求扰动（6个）：0.7x, 0.8x, 0.9x, 1.0x, 1.1x, 1.3x
```

## 分析输出指标

每个场景自动求解后，记录：

| 指标 | 含义 |
|------|------|
| 总成本 | 目标函数值 |
| 非标采购占比 | 非标采购量 / 总采购量 |
| 综合损耗率 | 总损耗 / 总采购量 |
| 求解状态 | Optimal / Feasible / Infeasible |

> [!success] 关键发现
> - 非标溢价在 ==10% 以内==时，非标采购仍比标准采购经济（因损耗节约大于溢价）
> - MOQ 降至 ==5 吨==时，非标占比从 55% 跃升至 75%（更多聚合组达标）
> - 需求缩放对总成本近似线性影响，方案结构稳定

## 在模型中的实现

```python
# solve_steel.py 中的敏感性分析循环
scenarios = []
for premium in [0, 0.02, 0.05, 0.08, 0.10, 0.15, 0.20]:
    for moq in [4, 5, 6, 8, 10, 12]:
        for scale in [0.7, 0.8, 0.9, 1.0, 1.1, 1.3]:
            result = solve_with_params(
                nst_premium=premium,
                moq_threshold=moq,
                demand_scale=scale
            )
            scenarios.append(result)

# 结果写入 Excel "敏感性分析" Sheet
write_sensitivity_results(ws, scenarios)
```
