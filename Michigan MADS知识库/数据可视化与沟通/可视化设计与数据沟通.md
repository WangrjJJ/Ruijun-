## 核心知识一：可视化设计框架

### 领域任务 vs 抽象任务

==领域任务 (Domain Task)== 是用户在具体业务场景中的目标；==抽象任务 (Abstract Task)== 是将其转化为可视化操作的通用描述。

> [!example] 芝加哥交通分析 (S522 HW1)
> - 领域任务："可视化不同路段一天中不同时段的车速"
> - 抽象任务："按分类变量（路段）和时间变量分组，用颜色编码一个定量变量"
> - 领域任务："比较南北向交通模式差异"
> - 抽象任务："按分面变量比较两组定量数据的分布"

### Munzner可视化分析框架

四层嵌套模型（由外到内）：

1. **领域问题描述** — 理解用户需要什么
2. **数据/任务抽象** — 将领域映射到数据类型和任务类型
3. **视觉编码/交互** — 选择标记和通道
4. **算法** — 高效渲染实现

## 核心知识二：Altair声明式可视化

### 核心语法结构

Altair基于==Vega-Lite==，采用"数据→编码→标记"的声明式模式：

```python
alt.Chart(df).mark_bar().encode(
    x=alt.X('variable:Q', scale=alt.Scale(domain=[0,80])),
    y='count()',
    color='category:N'
).properties(title='标题', width=400, height=300)
```

### 数据类型标记

| 标记 | 类型 | 示例 |
|------|------|------|
| `:Q` | Quantitative（定量） | 速度、温度 |
| `:N` | Nominal（名义） | 方向、类别 |
| `:O` | Ordinal（有序） | 月份、等级 |
| `:T` | Temporal（时间） | 日期、时间戳 |

### 常用图表模式

| 图表类型 | 标记 | 适用场景 |
|---------|------|---------|
| 柱状图 | `mark_bar()` | 分类比较 |
| 散点图 | `mark_circle()` | 二维关系 |
| 折线图 | `mark_line()` | 时间趋势 |
| 热力图 | `mark_rect()` | 二维分类交叉 |
| 面积图 | `mark_area()` | 密度/堆叠 |
| 规则线 | `mark_rule()` | 均值/阈值标记 |

### 图表组合

```python
# 层叠 (overlay): 同一画布叠加多个标记
chart = density_chart + mean_rule_chart

# 垂直拼接 (vertical concat)
combined = chart1 & chart2 & chart3

# 水平拼接 (horizontal concat)
combined = chart_left | chart_middle | chart_right

# 分面 (facet)
chart.encode(facet='DIRECTION:N')
```

> [!tip] Altair大数据限制
> Altair默认限制5000行数据。超过时需 `alt.data_transformers.disable_max_rows()` 或使用 `alt.data_transformers.enable('json')` 将数据存为中间JSON文件。

## 核心知识四：不确定性可视化

### Bootstrap采样分布

> [!example] 手机使用频率估计 (S524 Assignment 1)
> 总体：5000名学生（泊松分布 $\lambda=45$）。样本：30人。Bootstrap抽样 $B=5000$ 次构建均值的抽样分布。
> - 总体均值：45.06
> - 样本均值：44.33
> - Bootstrap均值：44.36（接近样本均值，验证了Bootstrap的无偏性）

Bootstrap步骤：

$$\text{for } b = 1, \dots, B: \quad \bar{x}^*_b = \text{mean}(\text{sample}(X, n, \text{replace}=\text{True}))$$

```python
bootstrap_means = []
for _ in range(B):
    sample_n = np.random.choice(sample_pickups, size=30, replace=True)
    bootstrap_means.append(sample_n.mean())
```

### 大世界 vs 小世界不确定性

| 类型 | 含义 | 示例 |
|------|------|------|
| ==小世界不确定性== | 模型内参数估计的不确定性 | 样本均值的标准误、系数的置信区间 |
| ==大世界不确定性== | 模型假设本身是否正确 | OLS线性假设、等方差假设 |

### 回归系数的Bootstrap

> [!example] 汽车油耗预测 (S524)
> 用 `hp` 和 `wt` 预测 `mpg` 的线性回归。对 $\theta_{wt}$ 做5000次Bootstrap：
> - 均值：$-3.86$（每增加1000磅，mpg下降3.86）
> - 标准差：$0.71$
> - Bootstrap分布近似正态，支持参数推断

```python
for _ in range(B):
    df_sample = cars_df.sample(n=k, replace=True, random_state=rng)
    model = LinearRegression().fit(df_sample[['hp','wt']], df_sample['mpg'])
    bootstrap_theta_wt.append(model.coef_[1])
```

## 核心知识六：数据沟通原则

### 技术报告结构

1. **Executive Summary** — 1段概述结论与建议
2. **问题定义** — 业务背景与分析目标
3. **数据与方法** — 数据来源、处理流程、模型选择
4. **发现** — 核心结果（图表为主，文字解释为辅）
5. **建议与局限** — 可行动的建议 + 诚实的局限性

### 可视化设计原则

- ==数据墨水比== (Data-Ink Ratio)：最大化数据信息与墨水量的比例
- ==最小惊讶原则==：图表的视觉映射应符合直觉（如红=危险/高值）
- 避免3D图表、倾斜饼图等失真编码
- 考虑色觉障碍：使用色觉安全调色板

## 方法选择指南

```
选择图表类型：
  ├── 比较类别 → 柱状图 / 点图
  ├── 时间趋势 → 折线图
  ├── 分布形状 → 直方图 / KDE密度图
  ├── 二维关系 → 散点图 (+ 回归线)
  ├── 二维分类交叉 → 热力图
  └── 不确定性 → Bootstrap密度图 / 误差棒

选择可视化库：
  ├── 快速探索 → Altair (声明式, 交互友好)
  ├── 精细控制 → Matplotlib (命令式, 出版级)
  ├── 统计图表 → Seaborn (高级统计可视化)
  └── 交互仪表板 → Altair + Panel / Streamlit
```

## 实战洞察

- Altair的 `transform_density` 和 `transform_regression` 在前端计算，大数据集会卡顿，建议先用Pandas预聚合
- Bootstrap抽样分布的标准差等于样本均值的标准误 $SE = s / \sqrt{n}$，这是理论与实验的一致性验证
- 热力图中用 `:O`（有序型）而非 `:Q`（定量型）编码月份，避免月份被当作连续值插值
- 数据沟通的黄金法则：先说结论，再说证据，最后说方法
- "大世界不确定性"提醒我们：所有模型都是错的，关键是知道错在哪里
- 可视化不是"画完即止"，需要配合文字注释解释"看到什么"和"意味着什么"
- 交通数据分析中，`-1` 值表示数据缺失而非负速度，预处理时必须过滤
