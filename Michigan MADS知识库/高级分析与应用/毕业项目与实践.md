## 核心知识二：探索性数据分析 (EDA) 模式

### EDA核心步骤

1. **数据概览**：`df.shape`, `df.dtypes`, `df.describe()`, `df.info()`
2. **缺失值分析**：缺失比例、缺失模式（随机/非随机）
3. **分布检查**：直方图、箱线图、偏度/峰度
4. **相关性分析**：散点矩阵、相关系数热力图
5. **异常值检测**：IQR法则、Z-score

### EDA速查模式

```python
# 数据概览三件套
print(df.shape)
print(df.dtypes)
df.describe().round(1)

# 缺失值可视化
df.isnull().sum().sort_values(ascending=False).head(10)

# 分布快速检查
df.hist(figsize=(15, 10), bins=30)

# 相关性矩阵
df.corr().style.background_gradient(cmap='coolwarm')
```

> [!tip] EDA黄金法则
> 在写任何一行模型代码之前，必须完成EDA。EDA的目标不仅是"了解数据"，更是发现==数据中隐藏的建模假设违反==（如非线性关系、极端偏态、多重共线性）。

## 核心知识三：复购预测项目 (Milestone II)

### 问题定义

==目标==：预测电商用户在促销活动后是否会成为复购买家（二分类问题）。评估指标：ROC-AUC。

### 数据来源与特征

| 数据表 | 内容 | 关键字段 |
|--------|------|---------|
| 用户信息 | 用户属性 | age_range, gender |
| 用户行为日志 | 浏览/收藏/加购/购买 | action_type, time_stamp |
| 商品信息 | 商品属性 | cat_id, brand_id |
| 商家信息 | 店铺属性 | merchant_id |

### 特征工程策略

| 特征类别 | 示例特征 | 构建方法 |
|---------|---------|---------|
| ==用户行为统计== | 总浏览数、购买转化率 | `groupby('user_id').agg()` |
| ==时间特征== | 最近一次购买距今天数 | `max(date) - last_purchase_date` |
| ==交互特征== | 用户-商家购买频次 | `groupby(['user_id','merchant_id']).count()` |
| 商品偏好 | 最常购买品类 | `mode()` / `value_counts().head()` |
| 行为序列 | 浏览→购买转化路径长度 | 行为日志时间排序后计算 |

> [!example] 特征工程关键洞察
> - ==行为比率特征==比绝对数量更有预测力（如"购买/浏览比"优于"总购买数"）
> - 时间窗口特征（近7天/30天/90天）捕获用户活跃度趋势
> - 用户-商家交叉特征是复购预测的核心（反映忠诚度）

### ML流水线设计

```
原始数据 → 数据清洗 → 特征工程 → 特征选择 → 模型训练 → 评估
  │            │           │           │          │        │
  └─缺失值     └─类型转换   └─聚合统计   └─相关性   └─交叉验证 └─AUC
    异常值       编码         比率特征     VIF       网格搜索    混淆矩阵
```

### 模型选择与调优

| 模型 | 优势 | 劣势 | AUC参考 |
|------|------|------|--------|
| LogisticRegression | 快速、可解释 | 线性假设 | ~0.62 |
| RandomForest | 特征交互、鲁棒 | 慢、需调参 | ~0.65 |
| XGBoost/LightGBM | ==高性能、处理缺失值== | 过拟合风险 | ~0.68 |
| 模型融合 (Ensemble) | 提升泛化能力 | 复杂度增加 | ~0.69+ |

> [!tip] 项目最终成绩
> 复购预测最终AUC达到0.6833，采用梯度提升树模型。关键经验：特征工程贡献了约80%的性能提升，模型选择和调参贡献约20%。

## 核心知识四：时间序列处理

### 时间序列分解

$$Y_t = T_t + S_t + R_t$$

- $T_t$：趋势分量（长期方向）
- $S_t$：季节分量（周期性模式）
- $R_t$：残差分量（随机波动）

### 行为日志时间特征

```python
# 时间戳解析
df['timestamp'] = pd.to_datetime(df['time_stamp'])
df['hour'] = df['timestamp'].dt.hour
df['dayofweek'] = df['timestamp'].dt.dayofweek

# 距离事件的天数
df['days_to_event'] = (event_date - df['timestamp']).dt.days

# 滚动窗口统计
df.set_index('timestamp').rolling('7D')['action'].count()
```

## 核心知识五：定性研究方法

### 定性 vs 定量研究

| 维度 | 定性研究 | 定量研究 |
|------|---------|---------|
| 目标 | 理解"为什么"和"怎样" | 量化"多少"和"什么关系" |
| 数据 | 访谈、观察、文档 | 数字、统计 |
| 分析 | 编码、主题分析 | 假设检验、建模 |
| 样本 | 小样本、目的性抽样 | 大样本、随机抽样 |
| 产出 | 概念框架、理论 | 统计结论、预测模型 |

### 定性研究流程

1. **研究设计**：确定研究问题、选择方法论（扎根理论/现象学/案例研究）
2. **数据收集**：半结构化访谈、参与式观察、文档分析
3. **编码 (Coding)**：
   - ==开放编码==：逐行标注概念
   - ==轴心编码==：发现概念间关系
   - ==选择性编码==：形成核心范畴
4. **主题分析**：从编码中提炼主题 (Themes)
5. **理论构建**：基于主题构建解释框架

> [!tip] 定性研究在DS中的价值
> - 需求发现阶段：通过访谈理解用户真实痛点（而非假设痛点）
> - 结果解释阶段：用定性方法解释为什么模型在某些场景下失效
> - 伦理评估阶段：理解算法对不同利益相关者的影响

### 混合方法研究

==混合方法==结合定性与定量：

- **解释性顺序设计**：先定量建模 → 再定性解释异常结果
- **探索性顺序设计**：先定性访谈 → 再定量验证假设
- **并行设计**：同时进行定性和定量，交叉验证

## 方法选择指南

```
项目初期：
  ├── 理解业务 → 利益相关者访谈 (定性)
  └── 理解数据 → EDA (定量)

建模阶段：
  ├── 基线模型 → LogisticRegression
  ├── 性能提升 → XGBoost/LightGBM
  ├── 特征工程 → 聚合统计 + 时间窗口 + 交叉特征
  └── 模型融合 → Stacking / Blending

评估阶段：
  ├── 二分类 → AUC-ROC, F1-Score
  ├── 不平衡数据 → Precision-Recall曲线
  └── 业务价值 → 与基线相比的增量收益
```

## 实战洞察

- ==特征工程是竞赛和项目的核心竞争力==，模型调参的边际收益远小于好的特征
- 复购预测中，用户-商家的交互特征（如历史购买频次、最近购买间隔）是最强特征
- 数据泄露是毕业项目中最常见的致命错误：确保特征只使用目标事件之前的数据
- 定性研究不是"软"方法，编码过程的严谨性等同于数据清洗的严谨性
- 项目报告中，==一张好的可视化 > 一页文字描述==
- Milestone I的主要价值是培养"数据直觉"，即通过EDA形成对数据的深层理解
- 混合方法研究在产品分析中越来越重要：AB测试（定量）+ 用户访谈（定性）
- 可重现性不是"加分项"而是"基本要求"，固定随机种子应成为肌肉记忆
