# 监督学习

> 监督学习是机器学习的核心分支，通过已标注数据学习输入到输出的映射。本笔记涵盖 SIADS 532 Data Mining I 与 SIADS 542 Supervised Learning 的核心内容，包括==回归==、==分类==、==模型评估==与==超参数调优==。重点关注模型复杂度与泛化性能之间的权衡。

## 核心知识：回归模型

### 多项式回归 (Polynomial Regression)

通过 `PolynomialFeatures` 将特征扩展为高阶多项式，本质仍是线性回归：

$$y = a_0 + a_1 x + a_2 x^2 + \cdots + a_n x^n$$

> [!tip] 模型复杂度与泛化
> 在 S542 Assignment 2 中，degree=1 导致==欠拟合== ($R^2_{test}=0.37$)，degree=3 为==最佳泛化== ($R^2_{test}=0.87$)，degree=11 导致==过拟合== ($R^2_{test}=0.78$)。这是偏差-方差权衡的经典案例。

```python
from sklearn.preprocessing import PolynomialFeatures
poly = PolynomialFeatures(degree=3)
X_poly = poly.fit_transform(X_train)
```

### 正则化回归

| 方法 | 惩罚项 | 特点 | sklearn类 |
|------|--------|------|-----------|
| Ridge | $\lambda \sum \beta_j^2$ | L2，系数收缩 | `Ridge` |
| Lasso | $\lambda \sum \|\beta_j\|$ | L1，特征选择 | `Lasso` |
| ElasticNet | L1 + L2 混合 | 两者结合 | `ElasticNet` |

> [!example] Lasso 恢复真实模型
> S542 中 Lasso (`alpha=0.01, max_iter=10000`) 在无噪声测试集上 $R^2$ 最高的 degree=3，成功恢复了真实的三次多项式结构。Lasso 通过将无关高阶系数压为零实现==稀疏解==。

### K-近邻回归 (KNN Regressor)

非参数方法，默认 $k=5$，适用于非线性关系：

```python
from sklearn.neighbors import KNeighborsRegressor
knn = KNeighborsRegressor().fit(X_train, y_train)
```

## 核心知识：分类模型

### 支持向量机 (SVM)

SVC 使用 RBF 核时，关键超参数为 `gamma`（核宽度）和 `C`（正则化强度）：

$$K(x, x') = \exp\left(-\gamma \|x - x'\|^2\right)$$

> [!tip] Gamma 与拟合状态
> S542 中 `validation_curve` 实验显示：`gamma=1e-7` 欠拟合（训练/测试准确率都低），`gamma=1e-4` 最佳泛化，`gamma=1e-2` 过拟合（训练 100% 但测试仅 63%）。

```python
from sklearn.svm import SVC
from sklearn.model_selection import validation_curve
train_scores, test_scores = validation_curve(
    SVC(random_state=0), X, y,
    param_name="gamma",
    param_range=np.logspace(-7, -2, 6), cv=3)
```

### Logistic 回归

$$P(y=1|x) = \frac{1}{1 + e^{-(\beta_0 + \beta^T x)}}$$

- 支持 L1/L2 正则化（`penalty='l1'` 需 `solver='liblinear'`）
- 输出概率：`predict_proba()` 用于阈值调整和 ROC/PR 曲线

### 决策树与集成方法

| 方法 | 核心思想 | 关键参数 |
|------|----------|----------|
| Decision Tree | 递归分裂，信息增益/基尼系数 | `max_depth`, `min_samples_leaf` |
| Random Forest | Bagging + 特征随机 | `n_estimators`, `max_features` |
| Gradient Boosting | 残差逐步拟合 | `learning_rate`, `n_estimators` |
| AdaBoost | 加权样本串行训练 | `n_estimators`, `learning_rate` |

## 核心知识：模型评估

### 分类评估指标

| 指标 | 公式 | 适用场景 |
|------|------|----------|
| Accuracy | $\frac{TP+TN}{TP+TN+FP+FN}$ | 类别平衡时 |
| Precision | $\frac{TP}{TP+FP}$ | 关注假阳性（如垃圾邮件） |
| Recall | $\frac{TP}{TP+FN}$ | 关注假阴性（如疾病筛查） |
| F1 | $\frac{2 \cdot P \cdot R}{P + R}$ | 精确率与召回率权衡 |
| AUC-ROC | ROC 曲线下面积 | 整体判别能力 |

> [!example] 垃圾邮件场景的 Precision 优先
> S542 Assignment 3 中，垃圾邮件检测的==误报==（好邮件标为垃圾）代价极高，因此优化目标为 Precision。Dummy 分类器 Precision 仅 0.40，而 SVC 达到 0.93，LogisticRegression GridSearchCV 优化 Precision 最高达 0.93。

### 混淆矩阵与阈值调整

```python
y_score = clf.decision_function(X_test)
y_pred = np.where(y_score > threshold, 1, 0)
confusion_matrix(y_test, y_pred)
```

> [!tip] 决策阈值的灵活性
> 调低阈值（如 -100）会使更多样本被预测为正类，提高 Recall 但降低 Precision。这体现了 ==Precision-Recall 权衡==。

## 核心知识：交叉验证与超参数调优

### GridSearchCV 模式

```python
from sklearn.model_selection import GridSearchCV
grid = GridSearchCV(
    LogisticRegression(solver='liblinear', random_state=42),
    param_grid={'penalty': ['l1','l2'],
                'C': [0.005, 0.01, 0.05, 0.1, 1, 10]},
    cv=5, scoring='precision')
grid.fit(X_train, y_train)
```

### 数据预处理要点

1. ==先拆分，再标准化==：避免测试集信息泄露到训练集
2. 正则化模型（SVM、Logistic Regression）==必须==做特征归一化
3. S542 实验表明归一化后 GridSearchCV 最优 Precision 为 0.931，未归一化为 0.924

## 方法选择指南

| 场景 | 推荐方法 | 理由 |
|------|----------|------|
| 小样本 + 高维 | SVM (RBF) | 核技巧处理非线性 |
| 需要概率输出 | Logistic Regression | `predict_proba` |
| 需要特征重要性 | Random Forest / GBDT | `feature_importances_` |
| 需要可解释性 | Decision Tree / Lasso | 稀疏结构 |
| 基线对比 | DummyClassifier | `strategy='most_frequent'` |

## 实战洞察

- 多项式回归中 degree 的选择直接决定欠拟合/过拟合，通常配合交叉验证选择
- Lasso 在高阶多项式中自动将无关特征系数置零，是天然的特征选择器
- SVM 的 `gamma` 参数对性能极敏感，必须通过 `validation_curve` 或 `GridSearchCV` 调优
- 垃圾邮件检测等不对称代价场景，应以 Precision 而非 Accuracy 作为优化目标
- Dummy 分类器的 `most_frequent` 策略永远预测多数类，Precision=0 且 Recall=0（对少数类）
- 特征归一化对正则化方法的影响虽小但真实存在（约 0.7% 差异），规范流程不可省略
