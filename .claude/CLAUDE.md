## Frontmatter 约定

所有笔记使用统一属性，便于 .base 过滤和智能体检索：
- `type`: MOC / 战略框架 / 重点行动计划 / 经营数据 / 会议纪要 / 概念笔记 / 项目实战 / 经营回顾 / 战略发展 / 能力建设
- `status`: 生效中 / 执行中 / 已完成 / 已更新 / 已归档
- `priority`: P0 / P1 / P2
- `owner`: 责任人姓名
- `policy_nr`: 方针编号（1~19）
- `tags`: 用于交叉检索

## 更新操作规范

1. **增量更新**：只修改变化的笔记，不要重新生成整个知识区
2. **读前查属性**：用 Grep 搜 frontmatter 中的 status/date 判断是否需要更新，不要全量读取
3. **会议纪要模板**：直接按 `会议纪要/` 下现有格式追加，frontmatter 含 date/host/attendees
4. **经营数据**：按季度新建（Q2/Q3/Q4），不覆盖历史
5. **Git 同步**：每次修改后 `git add 具体文件 && git commit && git push`
