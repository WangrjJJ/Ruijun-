## 操作规范

1. **增量更新**：只改变化的笔记，不重新生成整个知识区
2. **读前查属性**：Grep frontmatter 中 status/date 判断是否需要更新
3. **会议纪要**：按 `会议纪要/` 下现有格式追加，frontmatter 含 date/host/attendees
4. **经营数据**：按季度新建（Q2/Q3/Q4），不覆盖历史
5. **Git 同步**：每次修改后 `git add 具体文件 && git commit && git push`
