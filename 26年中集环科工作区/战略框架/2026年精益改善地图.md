# 2026年精益改善地图（第一性原理版）

> [!abstract] 定位与方法
> 本文档从**第一性原理**出发，以不可改变的结构性事实为起点，逐层推导精益可以改变的能力变量，收敛为2026年精益方针。
>
> 推导链：**结构性事实 → 可改变变量 → 精益战场 → 方针 → 举措 → 机制**
>
> 数据来源：ITCO全球罐箱报告 + 行业五力分析 + Q1经营实绩 + 赛意灯塔工厂诊断 + 各车间/部门一线调研反馈。
>
> 与 [[固本—精益制造]]（车间施策明细）形成 **"地图→行动"** 两层结构。

---

## 一、第一层：不可改变的结构性事实

> 第一性原理的起点——先列出你无法改变的东西。这些是约束条件，不是问题。

### 事实1：行业产能严重过剩，且是结构性的

| 指标 | 数据 | 来源 |
|------|------|------|
| 全球制造产能 | ~12万台/年 | 行业研究 Ch.2 |
| 2025年实际产量 | 28,521台（15年最低） | ITCO 2025 |
| 产能利用率（全球） | ~24% | 行业研究 Ch.2 |
| 中集环科Q1利用率 | 38%（高于行业但仍远低于盈利线） | Q1经营数据 |
| 需求中枢变化 | 从~6万台/年降至 **~3万台/年新常态** | 行业研究 Ch.2.9 |
| 真正的需求拐点 | **2028-2030年**（2015-2018年投放的~20万台集中退役） | 行业研究 Ch.2.9 |

> [!danger] 关键判断
> **需求中枢下移是结构性的，不是周期性的。** Q1订单+204%不是"复苏信号"，是以价换量的价格战——隐含单价-27%，中集在用财务稳健性挤压竞争者现金流。
>
> → **精益不能以"提升产能利用率"为目标。产能是充足的，问题不在产能，在成本和柔性。**

### 事实2：标罐是同质化价格战，成本是唯一护城河

| 指标 | 数据 |
|------|------|
| 标罐签单价 | USD 1.38万（Q1新签）→ USD 1.31万（在手下均价） |
| 标罐Q1-2月利润 | **-1,581万元** |
| 竞争格局 | NT Tank/JJAP低价侵蚀，胜狮大通罐箱亏损转型ESS |
| 行业报告判断 | 中集在用规模和财务优势主动压价清场 |

> → **标罐竞争维度是成本——比对手更便宜还能活下去。标罐精益的唯一目标：可量化的单位成本下降。**

### 事实3：供应链议价权结构性弱势

| 物料 | 供应格局 | 实际议价空间 |
|------|---------|-------------|
| 316L不锈钢 | 宏观产能过剩（5,000万吨vs需求3,250万吨），但太钢/南钢按月**集中报价**，单次谈判空间小 | 议价空间不在"砍单价"，在**购买时机判断 + 数量决策** |
| 阀件 | Fort Vale/Perolo双寡头垄断，公司缺乏自有精密加工能力 | 靠集中订单谈判换取利润空间 |
| 封头 | 135OVC已全自制，V型防波板100%自制 | 自制率提升是已验证的降本路径 |

> → **采购精益的空间不在"砍价"，在时机与数量优化（MILP模型方向正确）。** 更根本的出路：关键部件自制率提升 + 材料利用率提升（每一吨钢多出产品）。

### 事实4：客户集中度下降，定制化加速

| 变化方向 | 信号 | 数据支撑 |
|----------|------|---------|
| 客户结构分散 | 货主自有罐箱从196,477→213,514台 | ITCO |
| 租赁商囤箱停止 | 闲置率~15%，CAPEX接近零 | 行业研究 |
| 订单碎片化 | 定制化要求增多，响应周期变短 | 各车间/部门一线调研反馈（数据后续补充） |

> → **品种更多、批量更小、交期更短是趋势。精益不仅要降成本，还要提柔性。**

### 事实5：ETO环境下的组织现实——工艺-生产脱节是结构性的

| 现实 | 描述 |
|------|------|
| 现场知识状态 | 大量技巧/诀窍，但处于**半封闭**状态，特罐/非标严重依赖经验 |
| 工艺管控边界 | 只能维持通用工艺标准（焊接等），订单级微观管控跟不上——因为订单频次高、差异变化快 |
| 闭环缺失 | 订单在生产的实际执行、变化没有从车间回流到设计和报价 |

> [!important] 关键判断
> 这不是人的问题，是**系统能力问题**。在高频定制环境下，工艺不可能对每个订单做微观指挥。精益要做的不是"让工艺管得更细"，而是**让现场有能力在通用标准框架内自主应对变化**——这正是TWI和DBS一线赋权的核心理念。
>
> → **精益的战场在"知识基础设施"：让半封闭的经验变成可获取、可传授、可迭代的组织能力。**

## 二、第二层：从事实推导精益的战场

### 2.1 "不可改变"与"可以改变"的分界线

```
      不可改变（约束）                        可以改变（精益战场）
┌──────────────────────────┐    ┌──────────────────────────────┐
│ 行业产能过剩（结构性）      │    │ 单位成本（材料利用率/转化效率）   │
│ 标罐定价权在买方            │    │ 质量成本（返工/报废/索赔）       │
│ 钢材/阀件供应定价模式      │    │ 响应速度（报价→交付周期）       │
│ 客户加速分散、定制化增加    │    │ 柔性（换型时间/多品种小批量应对） │
│ ETO模式不可简化             │    │ 知识可复用性（隐性→显性→传播）  │
│ 工艺无法对每个订单微观指挥   │    │ 管理体系（对标/激励/人才）       │
│ 2028年前需求不会真正回升    │    │ 关键部件自制+供应链纵向协同      │
└──────────────────────────┘    └──────────────────────────────┘
          ↓                                ↓
      接受为约束                         精益的四个战场
```

### 2.2 四个精益战场

**战场一：成本竞争力 — "活下去"**

在价格战中，单位成本每降1%，就多一份活下去的底气。精益打三个成本项：

| 成本项 | 占比（估） | 精益杠杆 | 潜在空间 |
|--------|:---:|---------|---------|
| 材料成本 | ~50% | 利用率提升（排样优化/缺陷预防/工艺降耗） | 损耗率6.79%每降1pp = 显著增利 |
| 转化成本 | ~25% | 标准作业消除动作浪费 + 辅材/能耗工位级原单位 | 工时-5~10%，辅材-5~10% |
| 质量成本 | 隐藏 | 防错Poka-Yoke + 工序内品质保证 → 减少返工/报废/索赔 | 错漏装-20%，外观≥90分 |

> OEE提升在产能过剩时的价值不在"多做"，在"用更少资源做同样的量"——同样的产量，设备开5天 vs 7天，省下的能耗和人工就是利润。

**战场二：柔性响应力 — "抢别人不敢接的订单"**

客户分散 → 批次更多、批量更小、品种更多样。谁能更快响应、小批量也赚钱，谁就能抢增量。

| 柔性维度 | 精益杠杆 | 目标 |
|----------|---------|------|
| 换型速度 | SMED（涂装换色/焊接换型） | 换型时间-30% |
| 多品种应对 | 模块化标准作业 + 单元化生产 | 特罐工时-10%，碳罐/小件在制品-30% |
| 多能工 | TWI-JI + 技能矩阵 | 碳罐多能工≥40% |
| 快速响应 | OTD全流程可视化 + 报价TPI | 报价周期-30%，OTD首次可度量 |

**战场三：知识基础设施 — "让经验从半封闭变成组织能力"**

这是对你"工艺-生产脱节是结构性问题"判断的直接回应。精益对策不是让工艺管得更细，而是建立 **知识显性化→标准化→传授→迭代** 的闭环：

```
师傅脑中的诀窍 → 视频化/标准化 → TWI-JI传授 → 多能工扩展 → 实践反馈迭代
                        ↑
                  工艺维持通用标准
                  车间补充岗位级操作诀窍库
```

| 知识类型 | 当前状态 | 精益方法 |
|----------|---------|---------|
| 通用工艺标准（焊接/涂装等） | 工艺部维护 | 保持 |
| 岗位级操作诀窍 | 老师傅脑中，半封闭 | 视频SOP + TWI-JI + 技能矩阵 |
| 订单级生产变化 | 没有回流闭环 | 生产反馈→设计/报价端的经验库 |

**战场四：管理基础设施 — "让改善从运动变成日常"**

前三项是"做什么"，这一项是"怎么让它们发生"。

| 机制 | 当前缺口 | 2026年目标 |
|------|---------|-----------|
| DBSO | 无推进机构 | 7月挂牌，3-5人，直报总经理 |
| MOR | 月度经营会无SDQCG议程 | 月度MOR正式运行，总经理主持 |
| Daily Management | 仅班组晨会雏形 | 三层会议标准化，KPI日/周暴露 |
| 激励机制 | 摊派≥15件/季度 | 带级津贴+Kaizen发表奖+车间排名 |
| 人才培养 | 缺TWI方法论 | 黄带→绿带→黑带三级 |

## 三、第三层：2026年精益方针

> 从事实→战场→方针，逐层收敛。

### 3.1 方针总纲

> [!important] 2026年精益方针
> **降本以生存，蓄能以等待。**
>
> 在市场结构性过剩的2026-2027年窗口期，聚焦**成本竞争力**（活下去）和**柔性+知识**（蓄能力），通过**管理基础设施建设**（DBSO/MOR/激励机制）让改善可持续，确保2028-2030年需求拐点到来时，公司的成本结构、响应速度、组织能力已经完成升级。

### 3.2 四项方针

| 方针 | 回答的问题 | 2026年成功标志 |
|------|-----------|---------------|
| **方针一：建基线** | 我们现在到底什么水平？ | ① 核心设备OEE基线建立 ② 瓶颈工序标准作业覆盖率≥50% ③ OTD端到端周期首次可度量 ④ 材料损耗/辅材消耗/质量成本基线建立 |
| **方针二：攻成本** | 在价格战中靠什么比对手更便宜？ | ① 材料利用率提升（损耗率从6.79%下降） ② 标罐关键工序工时-5~10% ③ 辅材/能耗原单位管理覆盖全部车间 ④ 错漏装-20%，外观≥90分 |
| **方针三：提柔性** | 客户越来越分散、定制越来越多，怎么应对？ | ① 换型时间基线建立→SMED改善 ② 碳罐/小件单元化生产试点 ③ 多能工≥40%（碳罐） ④ 报价周期-30%，OTD首次可度量 |
| **方针四：建机制** | 怎么让改善可持续，而不是一阵风？ | ① DBSO 7月挂牌运作 ② MOR月度运营回顾每月运行 ③ 精益带级激励机制发布 ④ 知识管理（视频SOP + TWI-JI试点）启动 |

## 六、下个时代的竞争力重点：行业趋势×精益启示

> 综合Obsidian行业研究17份文件、ITCO全球罐箱报告、2026年公开市场数据，提炼五个"元趋势"——每个趋势回答"客户要什么变了"以及"精益该做什么"。

### 6.1 元趋势一：从"制造产能"到"智能资产"

| 信号 | 数据 |
|------|------|
| 全球罐箱保有量增速 | 1.93%（ITCO 2026），历史最低 |
| 智能罐箱市场 | IoT传感器+AI预测性维护，CAGR>20% |
| 头部运营商动作 | Hoyer/Stolt自建IoT平台3-5年 |
| 客户采购逻辑转变 | 从"买一个铁箱子"到"买自带健康监测、可追溯、可预测维修的物流资产" |

**中集应有的定位**：行业研究已判明——做**Cummins模式**（卖发动机+OBD2开放接口），不做Geotab模式（车队管理SaaS）。运营数据是运营商的命脉不能碰，但**设备健康数据是制造商的天然主场**。

> **对精益的启示**：制造端积累的质量数据（焊缝参数/探伤结果/材料批次/检验记录）是智能罐箱的数据底座。今天精益改善中建立的**质量追溯能力=明天的数据资产**。TPI-07（数字-精益对齐）不应只是"先改善再买系统"，而是"改善过程产生的数据直接设计成可对外输出的格式"。

### 6.2 元趋势二：从"标罐走量"到"特种罐走利润"

| 信号 | 数据 |
|------|------|
| 标罐价格战 | Q1订单+204%但单价-27%，标罐亏损1,581万 |
| 电解液罐箱 | 中国电解液出货量+46.4%（2025），溶剂跨洋运输天然适合ISO罐箱 |
| 半导体PTFE内衬罐 | 湿电子化学品国产替代从37%→50%+，全球<5家能做PTFE内衬 |
| 食品级罐箱 | GB 44917-2024强制食用油专用容器，创造全新市场 |
| 行业研究判断 | 即使拿下全部增量，仅能维持当前产量（8,000-12,000台/年）——不是"增长"而是"稳态再平衡" |

> **对精益的启示**："提柔性"方针不是锦上添花，是**特种罐接单能力的硬约束**。柔性不够=不敢接特种罐订单=毛利永远锁死在5%。模块化设计（方针#17目标20%→60%）+SMED快速换型+单元化生产——这三件事直接决定公司能否从"标罐价格战"切换到"特种罐利润池"。

### 6.3 元趋势三：从"一次性卖箱"到"全生命周期服务"

| 信号 | 数据 |
|------|------|
| 中集后市场布局 | 2025.12成立中集赛维罐箱服务（南京）公司 |
| 行业并购 | Quantix收购G&W Tanks（2024.10），整合加速 |
| 后市场毛利率 | 30-40%（vs 标罐制造5-16%） |
| 客户诉求 | 一站式服务（堆场/清洗/检验/维修/翻新） |

> **对精益的启示**：精益的TPI课题直接支撑服务化转型。OTD可视化（TPI-01）→交期可承诺。质量追溯（MFG-04）→维修可预测。报价标准化（TPI-02）→服务可定价。**客户买的不是一个箱，是一个"交期可承诺+质量可追溯+服务可续约"的完整体验。** 精益就是把这个体验工程化。

### 6.4 元趋势四：从"人工经验"到"AI增强制造"

| 信号 | 数据 |
|------|------|

| 行业AI趋势 | AI辅助设计（参数推荐/合规自动匹配）+预测性维护 |

> **对精益的启示**：这是"知识基础设施"方针的技术支撑。老师傅的诀窍+AI焊接参数推荐=隐性知识规模化。精益的知识显性化（视频SOP+TWI-JI）正好为AI训练提供结构化数据。**今天做的每一个标准作业视频，都是明天的AI训练样本。**

### 6.5 元趋势五：法规驱动的新壁垒和新需求

## 八、行动依赖逻辑：什么必须先做，什么必须后做

> 从"Phase 0→1→2→3"的线性叙事升级为**有明确前置条件的依赖链**。每一个"为什么必须先做"都有可论证的逻辑。

### 8.1 核心依赖链（主线）

```
Phase 0：质量Pareto + 视频SOP + MILP偏差表 + Mini-Kaizen（精益推进小组启动）
    │  前置条件：零（仅需质量部Excel数据+手机+已有MILP模型）
    │  产出：三份事实分析+一个一线改善成功案例

    │
    ├──→ 月度经营会插入"成本损失TOP3"议程
    │       前置：Pareto分析完成
    │       逻辑：MOR改造必须有"东西可看"——否则只是换了个议程格式
    │
    ├──→ PSP讨论会（Phase 1入口）
    │       前置：至少有Pareto分析+经营会跑了1次+Mini-Kaizen案例
    │       逻辑：用数据而非意见对齐"真问题"认知
    │
    ├──→ DBSO挂牌（Phase 1）
    │       前置：PSP达到基本共识 OR 总经理在经营会上追问了一次
    │       逻辑：没有共识的DBSO=企管部一个人打空气
    │
    ├──→ MOR正式运行（Phase 1）
    │       前置：DBSO有基本编制+"成本TOP3"跑通了3次以上
    │       逻辑：MOR是DBSO的"舞台"——没有MOR，DBSO没有存在感
    │
    ├──→ 激励机制发布（Phase 2）

    │       逻辑：没有成功案例支撑的激励方案=纸上谈兵。必须先证明"改善有用"，再谈"怎么激励"

    └──→ 全面课题启动（Phase 3 / 2027 H1）
            前置：DBSO 2-3人+MOR稳定+激励已发布+OTD基线已有
            逻辑：资源和管理带宽不够，不能同时推进24个课题
```

**评估维度说明**：

| 维度 | 评分标准 |
|------|---------|
| **业务价值** | 对毛利率提升/交期缩短/质量改善的量化贡献大小。高=直接改善标罐毛利或特罐接单能力；中=改善间接影响利润；低=基础建设，长期见效 |
| **实施难度** | 所需额外人力+IT系统依赖+跨部门协调复杂度。低=1-2人可启动，无需IT投资；中=需3-5人协作或简单IT工具；高=需多部门联合+系统建设 |

**四象限行动策略**：

| 象限 | 策略 | 启动时机 | 典型课题 |
|------|------|---------|---------|
| **优先攻坚**（高价值+低难度）| 立即启动，Phase 0 | 现在 | 质量Pareto、Mini-Kaizen、MILP偏差表、成本TOP3 |
| **战略投资**（高价值+高难度）| Phase 1启动，分批推进 | PSP讨论会对齐后 | 搬迁精益设计、VSM、标准作业、OTD可视化 |
| **速赢储备**（低价值+低难度）| 机动安排，见缝插针 | 有人有时间时 | 财务月结、5S、数字-精益对齐 |
| **条件触发**（高难度）| 前置条件满足后启动 | 依赖就绪后 | 工艺-生产协同（需PSP对齐）、APS（需OTD基线） |

### 8.3 逐课题前置-后置关系

| 课题 | 必须前置 | 为什么 | 解锁的后置课题 |
|------|---------|------|--------------|
| **质量Pareto** | 质量部Excel | 不知道TOP3问题就无法设计防错方向 | MFG-04防错专项、PSP讨论会 |
| **视频SOP试点** | 1个愿意的老师傅 | 知识显性化的概念验证 | MFG-03标准作业推广、TWI-JI |
| **MILP偏差表** | 已有模型+采购部配合 | 先看到偏差有多大→再决定是否做TPI-04 | TPI-04采购时机优化 |
| **Mini-Kaizen** | 1个有意愿的班组长+他的痛点 | 第一个一线成功案例=打破"改善=行政任务"认知 | 改善文化扩散、激励方案 |
| **OTD可视化VSM** | 跨部门协作（PSP对齐后） | OTD需要营销+生产+采购跨职能配合 | TPI-02报价流程、交期承诺 |
| **特罐搬迁精益设计** | 无前置（独立窗口） | 施工图定稿前必须完成 | 锁定未来10年运营效率 |
| **激励方案发布** | 3个以上一线改善案例 | 先证明"改善有用"→再谈激励 | 人才培养、带级体系 |
| **TPM试点** | OEE基线（MFG-02）+DBSO有编制 | 需要人推+有数据 | 设备管理数智化 |

### 8.3 为什么PSP讨论会不能最先开

```
错误顺序：先开PSP讨论会（6月第一周）
    → 各部门争论"问题是什么"
    → "我觉得是产能不够" "我觉得是质量问题" "我觉得是人的问题"
    → 没有数据，共识无法达成 → 各自保留意见 → 精益变成企管部一个人的事

正确顺序：Phase 0先跑2-4周
    → 质量Pareto（有数据）:"我们每年花X万返工费在这三个问题上"
    → MILP偏差表（有数据）:"最优采购 vs 实际采购差了Y%"
    → Mini-Kaizen（有故事）:"张班长用2小时改了工具架，每天省了30分钟"
    → PSP讨论会 = 用事实说话 → 共识自然形成
```

> PSP的核心不是"agree on the problem"，而是**"see the same facts"**。Phase 0的使命就是制造这些facts。

## 九、破局策略：资源约束下的务实路径

> 最关键的约束：**公司人力资源偏紧，大家大多数精力被日常经营订单牵制。** 如果再加"精益任务"就是压死骆驼的最后一根稻草。必须从"不占用额外人力"的角度破局。

### 9.1 必须打破的死循环

```
日常救火占满所有人 → 没时间做改善 → 问题反复出现 → 更忙
        ↑                                            ↓
        └──────────────── 死循环 ────────────────────┘
```

打破方式：**不是"增加改善工作"，而是用改善替换掉一部分救火工作。** 节省出来的时间就是下一轮改善的投入。第一个改善课题必须满足：
- 几乎不占用额外人力——不搞"改善周"把车间骨干拉走两天
- 见效快——2-4周内能看到结果，让参与者自己相信"改善能省时间"
- 省出来的时间当事人能感受到——不是报表数字，是"少加了两个小时班"

### 9.2 人力资源与工具的现实约束

**人力资源现实**：

| 角色 | 状态 | 能投入精益的时间 |
|------|------|:---:|
| 总经理 | 被经营压力+外部事务占满 | 每月2小时（MOR） |
| 部门长/车间主任 | 日常运营+订单交付占满 | 每月4-8小时 |
| 班组长/一线骨干 | 被生产任务绑死 | 每天15分钟（DM站会）+ 偶尔Mini-Kaizen |
| 工艺/技术人员 | 被订单设计变更占满 | 很难抽出整块时间 |
| 企管部 | 四大支柱并行 | **可能是唯一能投入较多时间的人** |

**IT/AI/工具现实**：

| 工具 | 当前状态 | 对精益的可用性 |
|------|---------|:---:|
| MILP采购模型 | 已运行 | ✅ 可直接用于采购时机优化 |
| IoT平台 | 招标完成，未上线 | ❌ 短期内不可用 |
| MES/QMS | 立项中 | ❌ 2026年内不可用 |
| Python/Pandas | 企管部数据分析能力 | ✅ 可做数据分析 |
| LLM/AI | 企管部已在推动 | ✅ 可用于文档生成、知识提取、数据分析 |
| Excel/PPT | 公司标配 | ✅ 可视化看板的务实选择 |
| 手机视频 | 人人都有 | ✅ 知识显性化最简单的工具 |

### 6.3 四个"零资源启动"的切入点

#### 切入点一：精益推进小组可以独立启动的事

不需要任何人批准，不需要额外编制，本周就可以开始：

| 行动 | 做法 | 耗时 | 价值 |
|------|------|:---:|------|
| **质量投诉Pareto分析** | 拿到近12个月客户投诉Excel→Python+AI做Pareto→一张A4纸可视化 | 精益推进组4-6h | PSP讨论会最有说服力的开场材料 |
| **视频SOP试点** | 选一个关键工序→手机录10分钟老师傅操作→LLM自动生成文字步骤→老师傅review修正 | 精益推进组2h + 老师傅30min | 知识显性化从零到一，不花一分钱 |
| **MILP偏差表+闭环** | 模型最优计划 vs 实际采购月度对比→偏差>10%追溯原因 | 精益推进组2h + 采购部30min/月 | 给已有投入加闭环，ROI立见 |

#### 切入点二：把精益藏进现成会议，不新增任何会

| 现有活动 | 改造方式 | 额外时间 |
|----------|---------|:---:|
| 月度经营分析会 | 加一张"成本损失TOP3"可视化看板，3分钟过完 | 3分钟 |
| 班组晨会 | 加一个问题："昨天有什么浪费/障碍？"只记录不解决 | 2分钟 |
| 工艺部周会 | 加一个环节："这周哪个订单的设计变更在车间走样了？" | 5分钟 |
| 采购部月度会 | 加MILP偏差表review | 5分钟 |

> **核心原则：不新增会议，只在现有议程中插入一个问题。总增量时间 <15分钟/周/人。**

#### 切入点三：第一个Kaizen——找一个牢骚最多的班组长+让他烦了三个月的小问题

| 选择标准 | 为什么 |
|----------|--------|
| 选人：找那个"牢骚最多"的班组长 | 牢骚多 = 有改善意愿，只是没有渠道 |
| 选题：选他烦了三个月但没人管的琐碎问题 | "工具找不到""流转卡每次都填错""换型后第一件总是废"这种级别 |
| 方法：DBS Mini-Kaizen，就2小时 | 观察→找根因→试一个对策→看结果。车间内完成，不写报告 |
| 收获：让他自己在班组晨会说结果 | 一个人说"我昨天改了X，今天省了Y分钟"，比企管部发十份通知都管用 |

#### 切入点四：AI的务实角色——让愿意改善的人少花时间在"写"上

| AI用途 | 具体做法 | 省谁的时间 |
|--------|---------|:---:|
| 数据分析加速 | Python+LLM→质量Pareto、产能波动根因提示 | 精益推进组：2天→2小时 |
| 文档生成 | LLM→A3报告草稿、SOP初稿、提案模板 | 所有需要写报告的人：2小时→30分钟 |
| 知识提取 | 老师傅口述→LLM转结构化SOP→师傅修正 | 工艺部+车间：从"没法做"到"30分钟出初稿" |

### 9.4 分阶段实施路径

> 从"2周破冰"开始，而非"全年计划"开始。每个阶段的进行取决于上一个阶段产生了可见结果。

#### Phase 0：破冰（6月）——精益推进小组启动 + 最少协作

| 行动 | 执行 | 耗时 | 门槛 |
|------|------|:---:|------|
| 质量投诉Pareto分析 | 精益推进组+AI工具 | 4-6h | 拿到Excel数据 |
| 第一个视频SOP试点 | 精益推进组+1名老师傅 | 2.5h | 手机+LLM |
| MILP偏差表V1.0 | 精益推进组 | 2h | 已有模型 |
| 月度经营会插入"成本损失TOP3" | 精益推进组+总经理 | 会前沟通15min | 总经理点头 |
| 找到第一个有意愿的班组长 | 精益推进组 | 1h/周 | 去车间沟通 |

> Phase 0 总投入：精益推进组核心成员每周半天。其他人几乎为零。输出：三份有说服力的分析材料 + 一个改善试点候选人。

#### Phase 1：生根（7-9月）——DBSO最小化启动

| 前提条件 | 行动 |
|----------|------|
| Phase 0 的三份分析+视频SOP+MILP偏差表已产生可见结果 | DBSO 正式挂牌（即便只有1人） |
| 总经理在月度经营会上看到了"成本损失TOP3"并追问了一次 | MOR 正式运行（从现有会议改造，不新增） |
| 第一个 Mini-Kaizen 成功（一个班组长解决了一个问题）| 第一次 Kaizen 发表会（5分钟×3个案例，附在月度经营会后） |
| OEE基线有初步数据 | TPM 试点 1 个车间 |

#### Phase 2：扩散（10-12月）——让成功案例说话

| 前提条件 | 行动 |
|----------|------|
| Phase 1 有 3 个以上一线改善案例 | 激励机制发布（已经有成功案例可展示） |
| OTD 可视化 VSM 完成 | 第二个 TPI 课题启动 |
| 视频SOP 试点被认可 | 知识管理（视频SOP+TWI-JI）扩展到3个工序 |

#### Phase 3：常态化（2027 H1）

| 前提条件 | 行动 |
|----------|------|
| DBSO 有 2-3 人 | 全面推进课题清单 |
| 特罐搬迁精益设计完成 | 2028-2030 准备期建设 |

### 9.5 课题重排序：先做这5件

从"要做24件事"收缩为"先做这5件"，按投入最少 + 见效最快 + 能制造势头排序：

| 优先级 | 课题 | 额外人力 | IT依赖 | 如果失败了？ |
|:---:|------|:---:|------|------|
| **1** | 质量投诉Pareto分析 | 仅需精益推进组 | Python+LLM（已有） | 方向正确即可积累 |
| **2** | 第一个Mini-Kaizen | 1个班组长2h | 无 | 一个班组长的2小时 |
| **3** | 视频SOP试点 | 1名老师傅30min | 手机+LLM | 一次有益的尝试 |
| **4** | MILP偏差表+闭环 | 采购部30min/月 | 已有模型 | 已有投入，不增风险 |
| **5** | 月度经营会插入成本TOP3 | 仅议程修改 | Excel | 会前沟通15分钟 |

> 这五件事的共同特征：**极低启动成本，失败不会造成组织性损失。但若做成一件，就有了继续推进的势能和资格。**

## 十一、关键成功因素与风险

### 11.1 必须做对的三件事

1. **Phase 0 必须产出可见结果** — 如果6月份拿不出三份有说服力的分析和一个成功的Mini-Kaizen，后面所有DBSO/MOR/激励机制都无从谈起。**整个精益体系的第一块多米诺骨牌不是总经理的批准，而是推进组能否产出令人信服的事实基线。**
2. **第一个Mini-Kaizen必须来自一线，不是企管部** — 如果第一个成功案例是企管部做的分析报告，车间会说"那是你企管部的工作"。如果是某个班组长自己解决了一个困扰三个月的问题，其他班组长会问"他怎么做到的"。
3. **特罐搬迁的精益设计窗口不能错过** — 这是唯一需要行政力量推动的事项，与Phase无关。必须写入项目章程。

### 11.2 风险清单（按破局路径更新）

| 风险 | 概率 | 影响 | 应对 |
|------|:---:|:---:|------|
| Phase 0 被日常事务淹没，推进组没时间启动 | 高 | 致命 | 每周半天固定时间，纳入精益推进组工作计划 |
| Phase 0 产出质量不足以说服 | 中 | 高 | 先做"质量Pareto"——最容易出成果、最不需要协作 |
| Mini-Kaizen 找不到合适的"牢骚班组长" | 低 | 中 | 先从碳罐车间找——有多能工文化，更容易接受 |
| 一个月后没有可见效果，PSP讨论会被推迟 | 中 | 高 | **PSP讨论会不要在Phase 0之前开**——先产出再拉齐 |
| 总经理认为"这不是当前优先级" | 中 | 致命 | MOR改造只加3分钟议程，不是"新项目"而是"现有会议更好开" |

## 七、关键成功因素与风险

### 7.1 必须做对的三件事

1. **总经理亲自站台MOR** — 精益体系能否立住，唯一最关键变量。MOR变成企管部独角戏 = 3个月内空心化。
2. **DBSO独立的汇报线** — 必须直接向总经理汇报。挂靠企管部 → 非制造TPI推不动、车间视为行政任务。
3. **特罐搬迁的精益设计窗口** — 施工图定稿前完成VSM未来图+3P。错过=锁定未来10年低效布局。

### 7.2 风险清单

| 风险 | 概率 | 影响 | 应对 |
|------|:---:|:---:|------|
| PSP讨论后认知不一致 | 中 | 高 | 用本图事实数据为基线，数据说话而非意见 |
| 车间抵触"又一个新花样" | 中 | 中 | 先用1-2个速赢Kaizen让结果说话 |
| DBSO人才难以招聘 | 高 | 中 | 优先从内部有意愿的班组长选拔培养 |
| 特罐搬迁精益与进度冲突 | 中 | 高 | 写入项目章程作为前置条件 |
| 产能过剩导致"改善无用论" | 低 | 高 | 改善目标不是更多产出，是更低单位成本——这在任何时候都有用 |
| 集团ONE与DBS口径不一致 | 低 | 低 | ONE为考核框架，DBS为实施方法 |
