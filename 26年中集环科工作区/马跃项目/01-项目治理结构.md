# 马跃一号 — 项目治理结构

> 核心原则：**企管部牵头协调 + 职能部门专业主导 + 业务部门需求拉动**。融合不是企管部"替各部门做"，而是企管部"确保各部门在正确的时间做正确的事"。

## 一、三层治理架构

```
                    ┌─────────────────────────────┐
                    │  融合管理委员会（决策层）       │
                    │  季国祥(主任) + 张毅 + 分管副总 │
                    │  月度会议 + 重大决策审批        │
                    └─────────────┬───────────────┘
                                  │ 汇报 ↑ ↓ 授权
                    ┌─────────────┴───────────────┐
                    │  融合PMO（执行协调层）          │
                    │  企管部牵头：王瑞俊            │
                    │  周例会 + 进度追踪 + 跨组协调   │
                    └─────────────┬───────────────┘
                                  │ 协调 ↑ ↓ 执行
      ┌───────────────┬───────────┼───────────┬───────────────┐
      │               │           │           │               │
  ┌───┴───┐     ┌─────┴────┐ ┌───┴───┐ ┌────┴────┐    ┌────┴────┐
  │战略治理│     │运营融合组 │ │财务法务│ │人才文化组│    │IT/数字化│
  │  组   │     │(最大组)   │ │  组   │ │         │    │   组    │
  └───────┘     └──────────┘ └───────┘ └────────┘    └────────┘
```

## 二、各层职责与人员

### 2.1 融合管理委员会（决策层）

| 角色 | 人员 | 职责 |
|------|------|------|
| **主任** | 季国祥（总经理） | 重大决策审批、资源调配、对外代表 |
| **副主任** | 张毅（企管部分管领导） | 融合战略把关、跨部门冲突仲裁 |
| **委员** | 包秋婧（营销） | 客户衔接、订单不受影响 |
| **委员** | 刘建中（采购物流） | 供应链整合、采购协同 |
| **委员** | 陈晓春（技术） | 技术/质量体系整合 |
| **委员** | 林爱彬（生产/HSE） | 制造能力转移、HSE对标 |
| **委员** | 财务负责人 | 财务并表、资金管控 |
| **委员** | 组织发展负责人 | 组织架构、人才保留、文化融合 |

**议事规则**：
- 月度融合回顾会（MOR格式：上月亮点/差距/对策/下月重点）
- 重大决策（>500万支出、组织架构变更、高管任免）须委员会表决
- 紧急事项由主任+副主任双签即可先行执行，事后报备

### 2.2 融合PMO（执行协调层）— **企管部牵头**

| 角色 | 人员 | 核心职责 |
|------|------|----------|
| **PMO负责人** | ==王瑞俊== | ①百日计划编制与追踪 ②跨组协调与问题升级 ③融合KPI仪表板维护 ④周报/月报汇总 ⑤风险登记册管理 |
| **PMO支持** | 企管部1人（专职） | 会议组织、纪要、任务追踪系统维护 |
| **PMO支持** | 数字化组1人（兼职） | 数据看板、系统工具支撑 |

**PMO的"三不"原则**：
- **不替代**各部门做专业判断 → 部门是最懂自己领域的人
- **不绕过**工作组组长直接指挥组员 → 尊重职能汇报线
- **不过夜**处理跨组阻塞项 → PMO的响应速度决定融合速度

## 五、信息流转机制

```
现场问题/异常
    │
    ▼
工作组晨会（15min, 当日解决）
    │ 无法解决
    ▼
PMO周例会（升级+资源协调）
    │ 需决策
    ▼
委员会月度会（决策+授权）
```

**关键规则**：
- **绿色**（正常推进）：PMO周报记录即可
- **黄色**（偏差<2周, 有对策）：PMO升级给副主任知会
- **红色**（偏差>2周或无对策）：PMO立即升级给委员会主任

## 六、企管部在治理架构中的"枢纽"定位

企管部不是"替各部门干活的"，而是：

1. **节奏守护者** — 确保百日计划的每个时间窗口不被错过（类似Timer角色）
2. **信息枢纽** — 五个组的进度、风险、依赖关系，只在PMO这里汇聚成全局视图
3. **阻塞清除器** — 跨组的扯皮和推诿，PMO有责任48h内升级解决
4. **标准看门人** — DBS方法论/中集管理标准的落地质量，PMO负责把关

> 一句话：**企管部确保"对的事在对的时间被对的人做完"——而不是企管部把所有事做了。**
