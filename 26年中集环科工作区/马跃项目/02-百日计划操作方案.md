# 马跃一号 — 百日计划操作方案

> 基于DBS四阶段模型（Phase 0~4），融合已有"百日融合计划责任及进度表"任务，由企管部PMO视角重新编排为可追踪的周行动计划。

---

## 总览时间线

```
Phase 0           Phase 1   Phase 2        Phase 3                      Phase 4
签约前(6-12周)    Day1    前30天          30-100天                     100天+
─────────────────┬────────┬──────────────┬─────────────────────────┬──────────→
  战略治理+尽调   │ 亮相   │ 稳定+观察     │ 深度整合+释放协同          │ 常态化运营
  DBS方法论导入   │ 安定   │ 快速改善      │ 系统对接+能力转移          │ 文化沉淀
                  │        │              │                          │
  企管部角色：    │ 企管部 │ 企管部角色：  │ 企管部角色：              │ 企管部角色：
  Playbook编制    │ 角色： │ PMO运转启动   │ 跨组协调+风险管控          │ MOR机制固化
  PMO体系搭建     │ 协调者 │ Kaizen组织    │ 协同效应追踪              │ 复盘+Playbook修订
```

## Phase 0：签约前准备（W-12 ~ W-1）

> 目标：不容忍"签约当天才开始准备"。企管部在Phase 0的核心产出是**可操作的百日计划 + 已激活的治理架构**。

### 企管部 Phase 0 行动清单

| 周次 | 行动 | 交付物 | 涉及部门 |
|:---:|------|--------|----------|
| W-12 | 启动PMO筹备：确定企管部专职人员、建立项目管理工具（共享看板/任务追踪） | PMO工具就绪 | 企管部、数字化 |
| W-10 | 完成融合Playbook终稿（已完成 ✅） | 融合策划书V1.0 | 企管部 |
| W-8 | 召开第一次工作组组长预备会：明确各组百日目标+交付物标准 | 组长共识备忘录 | 五组组长 |
| W-6 | 完成Day 1准备清单（详见[[03-Day1准备清单]]）并分配责任人 | Day 1清单V1.0 | 全部门 |
| W-4 | 尽调信息汇总：从尽调团队获取标的运营数据基线（财务/质量/产能/人员/客户） | 数据基线报告 | 战略发展部、财务部 |
| W-3 | 确定派驻GM人选，完成GM的DBS速成培训（1周） | GM就位+培训完成 | 组织发展部、企管部 |
| W-2 | 委员会第一次正式会议：审批百日计划、确认预算、授权PMO | 委员会决议 | 委员会全体 |
| W-1 | Day 1演练：全流程桌面推演（谁在什么时间做什么、说什么） | 演练checklist | PMO+五组 |

> [!warning] Phase 0 最易被忽视的风险
> **IT/系统对接启动太晚**。ERP对接、OA导入、邮箱/域控创建等通常需要4-8周，如果在签约后才启动，会拖累整个运营融合进度。建议W-4即启动IT预备（在合规前提下）。

## Phase 1：Day 1（W0）

> 目标：**安定人心、传递信心、守住客户和供应商界面**。Day 1不是在"做事"，是在"做姿态"。

### Day 1 当日时间线

| 时间 | 事项 | 负责人 | 企管部角色 |
|------|------|--------|-----------|
| 08:00 | 法律交割完成确认 | 法务部 | 确认后发"Day 1启动"信号 |
| 09:00 | Town Hall全员大会 | CEO+派驻GM | 协调CEO出席、审阅演讲稿 |
| 10:30 | 中高层1on1见面会（派驻GM × 各部门负责人） | 派驻GM | 准备见面会提纲 |
| 13:00 | TOP20客户逐一电话/拜访 | 营销中心 | 确认客户名单、准备话术 |
| 13:00 | 核心供应商逐一通知 | 采购物流 | 确认供应商名单 |
| 15:00 | 印章/证照/密钥交接 | 法务+财务 | 监交 |
| 16:00 | 银行账户授权变更 | 财务部 | — |
| 17:00 | DBSO正式挂牌 + 第一个Kaizen计划公示 | 企管部(DBSO) | ==企管部主场== |
| 18:00 | Day 1总结短会（PMO+五组长，30min） | PMO | ==企管部主持== |

### 企管部 Day 1 核心动作

1. **审阅GM的Town Hall讲稿** — 确保三要素：肯定过去成就 + 不回避"会不会裁员/换人" + 明确100天的节奏
2. **DBSO挂牌** — 物理空间（一间会议室即可）+ 第一个Kaizen计划上墙（选最显眼的痛点）
3. **发放"员工一封信"** — 企管部起草，CEO+派驻GM联署，含Day 1 FAQs
4. **启动"问题热线"** — 临时邮箱/企业微信通道，收集员工疑问和不安信号

## Phase 2：前30天 — 稳定运营 + 快速观察（W1-W4）

> 目标：**不做大动作、不搞组织调整、不换人**。核心矛盾是"冲动做事"与"先看清楚"的平衡。

### 周计划

| 周次 | 企管部PMO驱动事项 | 各组关键动作 | 里程碑/交付物 |
|:---:|-------------------|-------------|--------------|
| **W1** | 启动每日晨会（15min站立会）；建立PMO仪表板初版 | 组织架构+高管任命发布；全员劳动合同签署启动 | PMO仪表板V1.0 |
| **W2** | 组织第一次VSM（价值流图）活动 — 选"订单到交付"主线 | 财务制度对接启动；采购供应商清单合并 | VSM现状图 |
| **W3** | 第一次Kaizen活动（3天）— 选产线最显眼的浪费点 | 质量体系双轨期启动（原体系+中集标准并行） | Kaizen成果+标准固化 |
| **W4** | 前30天总结报告（提交委员会）；制定30-100天修正计划 | 各工作组完成"30天发现报告" | 30天总结报告 |

### 企管部在Phase 2的三个"第一"

| 动作 | 为什么重要 | 怎么做 |
|------|-----------|--------|
| **第一个VSM** | 建立"以流程为中心"的共同语言，而非部门视角 | 选Perolo最核心产品的一条完整价值流，买卖双方同坐一张桌前画图 |
| **第一个Kaizen** | 让全员看到"融合不是来裁员的，是来让工作变轻松的" | 选最显眼、最烦人、最容易改的痛点（如工装夹具不合理、物料摆放混乱） |
| **第一次MOR** | 建立"数据说话、问题导向"的会议文化 | 30天总结按MOR格式：指标达成/差距/根因/对策，不允许只报喜 |

### 前30天禁止事项

- ❌ 大规模组织架构调整
- ❌ 更换中层管理人员（除非发现重大违规）
- ❌ 强制更换供应商
- ❌ 强推中集的管理制度替代原有制度
- ❌ 大面积IT系统替换

> "先观察，再判断，后行动。第一个月的任务是听和看，不是改。" — DBS融合第一原则

## Phase 3：30-100天 — 深度整合 + 释放协同（W5-W12）

> 目标：**每个工作组在100天窗口内至少完成一项"可量化的改善成果"**。从"观察模式"切换到"行动模式"。

### 五大工作组100天冲刺目标

| 工作组 | 100天必成目标 | 量化指标 | 企管部PMO追踪点 |
|--------|-------------|----------|----------------|
| **战略治理** | 董事会/股东会召开完毕，治理架构正式运行 | 工商变更完成日 | 章程/工商进度 |
| **运营融合** | ①采购协同首批落地（集采降价≥3%证伪或证实）②质量双轨期满，确定统一标准时间表 | 采购协同金额、PPM对比 | ==每两周Kaizen追踪== |
| **财务法务** | ①财务并表完成 ②资金管控体系运行 ③重大合同继承无遗漏 | 并表完成日、合同继承率100% | 周度财务对接进度 |
| **人才文化** | ①关键人才100%签署保留协议 ②薪酬对标完成 ③第一次文化融合工作坊完成 | 保留协议签署率、薪酬对标报告 | 关键人才状态每周更新 |
| **IT/数字化** | ①ERP基础对接（至少科目/物料主数据贯通）②OA/邮箱/域控上线 | 系统上线日、数据贯通率 | 系统上线倒计时 |

### W5-W12 企管部PMO周节奏

```
每周一 09:00  PMO周例会（五组长，60min）
             ├─ 上周红黄绿状态更新（5min/组）
             ├─ 跨组阻塞项清理（20min）
             └─ 下周关键里程碑确认（15min）

每两周       Kaizen活动（3-5天，DBSO组织）
             第1个Kaizen：产线效率
             第2个Kaizen：订单到交付流程
             第3个Kaizen：采购到付款流程

每月末       委员会月度MOR（90min）
             ├─ PMO汇总报告（王瑞俊，20min）
             ├─ 各组深度汇报（10min/组）
             └─ 决策+资源调配（20min）
```

### 企管部在Phase 3的关键产出

| 产出物 | 时间 | 内容 |
|--------|:---:|------|
| **协同效应追踪表** | W6起持续 | 每项协同项目：预期金额/实际/偏差/根因（详见DBS Playbook 7.2节） |
| **风险登记册更新** | 每周 | TOP10风险的状态更新、新增风险、已关闭风险（详见[[04-风险登记册]]） |
| **百日冲刺仪表板** | W8起 | 五组KPI红黄绿、里程碑达成率、阻塞项数量趋势 |
| **100天总结报告** | W12 | 提交委员会，作为Phase 4的基线 |

---

## Phase 4：100天+ — 常态化（W13起）

> 目标：**融合从"项目态"回归"运营态"**。PMO逐步退出，MOR机制承接。

| 周次 | 企管部PMO动作 |
|:---:|-------------|
| W13 | 100天总结报告提交委员会；组织第一次"融合复盘会"（各组回答：如果重来一次，哪些做对/哪些改） |
| W14 | PMO转入"轻量模式"：周会改双周会，仪表板改月更新 |
| W16 | DBSO角色从"推动者"转为"教练/支持"；派驻GM主导MOR |
| W20 | 更新Playbook（基于本次融合实战经验）；PMO正式解散，融合管理移交MOR机制 |

## 企管部百日工作量评估

| 阶段 | 企管部投入 | 关键约束 |
|------|:---:|------|
| Phase 0 (W-12~W-1) | **中高**（PMO筹备+Playbook+Day1清单+GM培训） | 需尽早确定企管部专职人员 |
| Phase 1 (Day1) | **极高**（全日在场协调） | Day 1当日不可分心 |
| Phase 2 (W1-W4) | **高**（晨会+Kaizen+MOR建立） | 前两周是机制建立窗口 |
| Phase 3 (W5-W12) | **中高**（周会+追踪+Kaizen） | 跨组协调量取决于阻塞项数量 |
| Phase 4 (W13+) | **中低**（复盘+过渡） | 移交MOR |

> [!warning] 资源保障要求
> 王瑞俊在百日期间需投入约**50-60%精力**于马跃项目。日常企管部工作（商业计划督办、精益、数字化）需明确优先级排序，必要时部分工作授权或延期。建议向张毅总确认：马跃项目是否为Q3企管部第一优先级。
