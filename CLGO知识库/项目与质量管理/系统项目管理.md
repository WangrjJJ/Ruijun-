# 系统项目管理

> [!abstract] 概述
> 系统讲解项目管理的核心方法论，涵盖==CPM关键路径法==、==CCM关键链法==、==DSM设计结构矩阵==，以NPI（新产品导入）ECU项目为大作业载体。

## 核心知识框架

### 项目管理铁三角

- **时间-成本-质量**: 三者相互制约，项目经理的核心任务是在约束下寻找最优平衡
- 项目管理本质：==协调而非执行==，项目经理是指挥家不是演奏者

### 关键路径法（CPM）

- 正推（最早开始/最早结束）+ 反推（最晚开始/最晚结束）
- ==关键路径==：总浮动时间=0的活动链，决定项目最短工期
- 非关键路径的浮动时间可用于资源调配

### 关键链法（CCM）

- 在CPM基础上考虑==资源约束==
- 项目缓冲（Project Buffer）置于关键链末端
- 汇入缓冲（Feeding Buffer）保护关键链
- 核心理念：消除个体活动安全余量，集中管理项目级缓冲

### 设计结构矩阵（DSM）

- 分析任务间的==依赖关系和信息流==
- 识别循环依赖（Coupled Tasks）→ 需要迭代
- 分区（Partitioning）→ 最优任务排序
- 撕裂（Tearing）→ 打破循环依赖

### 项目管理软件

- MS Project (.mpp) 用于进度编排和资源分配
- DSM v2.1 (Excel宏) 用于任务依赖分析
- Matlab脚本用于招投标分析（bid.m）

## 作业与实践要点

### 大作业: NPI ECU项目

- 新产品导入（ECU电控单元）的完整项目管理
- CPM与CCM方法的实际应用
- 项目任务分解、进度编排、资源分配
- 关键路径识别与风险缓冲设置

### 工具应用

- MS Project进行项目进度管理
- DSM矩阵分析任务依赖
- Matlab进行决策建模

## 与工作实践的关联
