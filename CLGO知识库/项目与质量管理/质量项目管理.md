# 质量项目管理

> [!abstract] 概述
> 聚焦质量问题的==系统化解决方法==，核心工具包括KT分析法（情境/原因/决策/潜在问题分析）和5 Why根因分析，强调结构化思维在质量改善中的应用。

## 核心知识框架

### KT分析法（Kepner-Tregoe）

| 阶段 | 目的 | 核心问题 |
|------|------|----------|
| **情境分析SA** | 理清问题全貌 | 发生了什么？涉及谁？影响范围？ |
| **原因分析PA** | 找到根本原因 | IS/IS NOT对比、差异分析、验证假设 |
| **决策分析DA** | 选择最优方案 | 必要条件/期望条件加权评分 |
| **潜在问题分析PPA** | 预防风险 | 可能出错什么？预防/应急措施 |

### 5 Why根因分析

- 从表面现象连续追问"为什么"直到触及==系统性根因==
- 通常3-5层即可到达可行动的根因层
- 每一层都需要==事实验证==，避免主观臆断
- 与鱼骨图（石川图）配合使用效果更佳

### 问题解决流程

```
定义问题 → 收集数据 → 原因分析 → 制定对策 → 实施验证 → 标准化
```
